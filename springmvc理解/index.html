SpringMVC理解

一、SpringMVC前奏

1.常用注解

@Controller,@RequestMapping,@Autowired,@RequestParam,@Service

2.简单代码演示

/**
     * 更新文章
     * @param article
     * @return
     */
    @RequestMapping("updatearticle")
    @ResponseBody
    public String updateArticle(Article article){
        String msg = "1";

        try {
            userService.updateArticle(article);
            msg = "0";
        }catch (Exception e){
            e.printStackTrace();
        }

        return msg;
    }

3.SpringMVC是啥？

SpirngMVC是由servlet包装而来，封装成了一个叫做DispatchServlet的类。

在web.xml中的配置如下：

<servlet>
        <servlet-name>boot-crm</servlet-name>
        <servlet-class>org.springframework.web.servlet.DispatcherServlet</servlet-class>
        <init-param>
            <param-name>contextConfigLocation</param-name>
            <param-value>classpath:spring/springmvc.xml</param-value>
        </init-param>
        <!-- 配置springmvc什么时候启动，参数必须为整数 -->
        <!-- 如果为0或者大于0，则springMVC随着容器启动而启动 -->
        <!-- 如果小于0，则在第一次请求进来的时候启动 -->
        <load-on-startup>1</load-on-startup>
    </servlet>
    <servlet-mapping>
        <servlet-name>boot-crm</servlet-name>
        <url-pattern>*.action</url-pattern>
    </servlet-mapping>

4.IOC容器简介

IOC容器实质上是一个Map，大致如下

Map iocMap = new HashMap();
OrderServlce orderService = new OrderService();
iocMap.put("orderService",orderService);

//使用的时候
@Autowired
//直接取出：iocMap.get("orderService");
private OrderService orderService;

5.Tomcat启动加载SpringMVC开发的war流程

Tomcat启动

①加载war包中的class文件

②创建IOC容器，即Map对象

③通过ScanbasePackage类：扫描war包下的@Controller，@Service注解的类

④实例化：将扫描到的类通过反射实例化，并放入IOC容器中（Map）

⑤依赖注入：将存在依赖的bean注入

⑥UrlMapping：http请求路径与Method建立映射关系

Tomcat工作

①发送http请求，调用servlet的doGet/doPost方法

②找到从UrlMapping中对应的method方法对象

③找到Method对象方法后，直接调用

④相应返回结果
